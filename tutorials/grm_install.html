Installation

  1. CMake installation
  2. Orfeo toolbox installation
  3. GRM installation
In the following, we consider we are using the Ubuntu Linux distribution (Sorry for those who use Mac, Windows or a Linux distribution different from Debian. For once, Linux is given priority.)
The minimum version accepted for gcc is 4.7 (you can check it in a terminal by invoking the command g++ --version) and the minimum version for clang is 3.2

CMake

To install the CMake software, you have two options:

  • You want to use cmake in command line and put your arguments manually.
  • You want a nice GUI
For the first option, execute the following command 
	
		sudo apt-get install cmake
For the second 
	
		sudo apt-get install cmake cmake-curses-gui

Orfeo Toolbox (otb)

For Ubuntu 12.04 and higher, OTB development libraries are available as Debian packages through APT repositories. Since release 3.14.1, they are available in the ubuntugis-unstable repository.
You can add it by using this command lines: 

	
		sudo aptitude install add-apt-repository  
		sudo apt-add-repository ppa:ubuntugis/ubuntugis-unstable
		sudo apt-get update
After you can run: 
	
		sudo apt-get install libotb-dev
Other useful OTB packages are available. For more information click here, but for our library this command is enough.

GRM installation

First you have to download the source code here
or if you have git installed you can run: 

	
		git clone http://tully.ups-tlse.fr/lassallep/grm/
Then I suggest you to go to the grm directory: 
	
		cd grm/
and create a build directory: 
	
		mkdir grm-build | cd grm-build
First you have to generate the Makefile with cmake: 
	
	cmake \
	-DCMAKE_CXX_FLAGS:STRING=-std=c++11 \
	-DOTB_DIR:PATH=/usr/lib/otb \
	-DITK_DIR:PATH=/usr/lib/cmake/ITK-4.6 \
	-DGDAL_CONFIG:FILEPATH=/usr/bin/gdal-config \
	-DGDAL_INCLUDE_DIR:PATH=/usr/include/gdal \
	-DGDAL_LIBRARY:FILEPATH=/usr/lib/libgdal.so \
	../src/
The previous commands works if you have installed the otb development libraries from the package as i described before (in a standard way). If you have used another way you have to give the right otb directory path (where the OTBCMake.config is located) and the right gdal paths.
Note also that c++11 is used which explains the minimum version required for gcc or clang.
Once the Makefile generated just run: 
	
	make
If everything is fine, you should have the GRM library in the sub-directory Library/ and the applications into Applications/
